VAI Pricing Agent

Pricing decision support for portfolio review, SKU diagnostics, and simulation.

The system combines workbook-based retail pricing data, business rules, simulation logic, and an analyst-facing web interface to help teams review pricing opportunities, understand recommendation drivers, and test changes before execution.

What The System Does

1. Loads pricing workbooks and normalizes SKU data

Excel sheets are parsed into a consistent record structure including price, cost, margin floor, competitor prices, KVI flag, inventory, and source workbook signals.

2. Generates recommendations at SKU level

The engine recommends increase, decrease, or hold by balancing market position, margin protection, demand sensitivity, and inventory context.

3. Simulates commercial outcomes

Users can see projected volume change, expected units sold, revenue impact, margin impact, and projected margin at any proposed price.

4. Supports analyst review and action

The frontend provides dashboard views, SKU detail pages, agent review prioritization, and an SKU Q&A interface for explanation and follow-up.

Primary Outcomes
  • Faster identification of margin risks and overpriced / underpriced SKUs
  • Consistent recommendation logic across the pricing queue
  • Transparent explanation of why a recommendation was made
  • Ability to test alternate prices before commercial sign-off
  • Portfolio-level visibility combined with SKU-level detail

Technical Detail

How the system is built and how the recommendation logic works.

The current implementation is a local full-stack application with a FastAPI backend, a React frontend, workbook parsing logic, a recommendation engine, and a simulation module that estimates the effect of price changes.

Architecture
Frontend React + Vite interface for dashboard, SKU queue, SKU detail, chat, and manual simulation.
Backend API FastAPI service that loads the active dataset, generates recommendations, and serves dashboard / SKU endpoints.
Data loader Workbook parsing logic that extracts supported sheets and maps raw columns into typed SKU records.
Decision engine Candidate-price evaluation using margin, market gap, revenue impact, margin-dollar impact, and business guardrails.
Simulation engine Rule-based elasticity model that estimates unit change and commercial outcomes at a proposed price.
Vector / LLM layer Local vector store for recommendation context plus optional LLM support for SKU-level question answering.
Current Logic Highlights
  • Demand elasticity is estimated from KVI status, promo status, stock cover, market gap, and source pricing flags.
  • The engine scores multiple candidate prices rather than applying a single fixed rule.
  • Recommendations are constrained by margin floor, market distance, and sensible move size.
  • Simulation outputs include rounded-up expected units sold and projected gross margin percentage.
  • Workbook-driven fields such as stock cover and inventory interpretation are now read from updated sheet formats.
Recommendation and Simulation Formula Summary
Expected Volume Change

Price change % × estimated demand elasticity

Expected Units Sold

Last week units × (1 + expected volume change), rounded up

Expected Revenue Impact

Projected revenue − current revenue

Expected Margin Impact

Projected gross profit − current gross profit

Projected Margin

(Proposed price − cost) / proposed price

Recommendation Choice

Best-scoring candidate after revenue, margin, market, and guardrail checks

Operational Benefits
  • Analysts can explain recommendations with evidence rather than static rules alone.
  • Pricing decisions are linked to commercial impact, not just competitor gap.
  • The interface helps teams move from portfolio review to SKU action without changing tools.
  • Workbook refreshes can be incorporated quickly through supported sheet mappings.
Current Scope and Next Enhancements
  • Rule-based elasticity can later be calibrated with historical price / sales data.
  • Confidence scoring can be refined using backtesting accuracy.
  • Executive reporting can be expanded with exportable summaries and approval workflows.
  • Recommendation governance can include category-specific rules and commercial override logging.
